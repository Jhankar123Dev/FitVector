## 2. Naming conventions

### 2.1 Files and folders
| Type | Convention | Example |
|------|-----------|---------|
| React components | kebab-case file, PascalCase export | `job-card.tsx` в†’ `export function JobCard()` |
| Pages | `page.tsx` inside route folder | `app/(dashboard)/jobs/page.tsx` |
| API routes | `route.ts` inside route folder | `app/api/jobs/search/route.ts` |
| Hooks | `use-` prefix, kebab-case | `use-user.ts` в†’ `export function useUser()` |
| Types | kebab-case file, PascalCase types | `job.ts` в†’ `export type Job = {...}` |
| Utilities | kebab-case | `format-date.ts` or grouped in `utils.ts` |
| Python files | snake_case | `scraper_service.py`, `ai_service.py` |
| Python classes | PascalCase | `class ScraperService:` |
| Python functions | snake_case | `def tailor_resume():` |
| Database tables | snake_case, plural | `users`, `job_matches`, `tailored_resumes` |
| Database columns | snake_case | `created_at`, `match_score`, `user_id` |
| Environment variables | SCREAMING_SNAKE_CASE | `CLAUDE_API_KEY`, `SUPABASE_URL` |
| CSS classes | Tailwind utilities only | No custom class names except in `globals.css` |

### 2.2 Code style
- **TypeScript:** strict mode enabled, no `any` types (use `unknown` + type guards)
- **React:** functional components only, no class components
- **Imports:** absolute imports via `@/` alias (maps to `src/`)
- **State management:** React Query (TanStack Query) for server state, Zustand for client state (if needed)
- **Forms:** React Hook Form + Zod validation
- **API responses:** always return `{ data, error, message }` shape

### 2.3 Git conventions
- **Branch naming:** `feature/job-search`, `fix/resume-pdf-crash`, `chore/update-deps`
- **Commit messages:** conventional commits: `feat:`, `fix:`, `chore:`, `docs:`, `refactor:`
- **PR titles:** same format as commits
- **Main branch:** `main` (production), `develop` (staging)

## 5. Component architecture principles

### 5.1 Component hierarchy
```
Page (server component вЂ” data fetching)
  в””в”Ђв”Ђ Client wrapper (client component вЂ” interactivity)
      в””в”Ђв”Ђ Feature component (job-list, kanban-board)
          в””в”Ђв”Ђ UI primitives (button, card, input from shadcn/ui)
```

### 5.2 Data fetching pattern
- **Server components** fetch data directly from Supabase (SSR)
- **Client components** use TanStack Query for client-side data + caching
- **Mutations** go through API routes в†’ validated в†’ proxied to Python if needed
- **Optimistic updates** for tracker actions (drag-drop, status changes)

### 5.3 Error handling pattern
```typescript
// Every API route follows this pattern:
export async function POST(req: Request) {
  try {
    const session = await getSession();
    if (!session) return Response.json({ error: "Unauthorized" }, { status: 401 });
    
    const body = await req.json();
    const parsed = schema.safeParse(body);
    if (!parsed.success) return Response.json({ error: parsed.error.flatten() }, { status: 400 });
    
    await checkUsageLimit(session.user.id, "resume_tailor");
    
    const result = await pythonClient.post("/ai/tailor-resume", parsed.data);
    
    return Response.json({ data: result });
  } catch (error) {
    if (error instanceof PlanLimitError) {
      return Response.json({ error: error.message, upgrade: true }, { status: 429 });
    }
    console.error(error);
    return Response.json({ error: "Internal server error" }, { status: 500 });
  }
}
```
